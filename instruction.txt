DocFinder Department Pages Development Guide
Welcome to the DocFinder frontend repository! This guide details the development of dynamic Department Pages (e.g., src/app/department/[slug]/page.jsx) in our Next.js application. Your existing src/app/layout.jsx (with its Header and Footer) will wrap these components.
Key Feature: The Departments link in your Header will present a submenu. Clicking any department (e.g., "Marketing & PR", "IT, Product, Data") will load a page with a consistent structure, but content tailored to that specific department.
DocFinder Department Pages aim to provide detailed insights into specific teams, their roles, culture, and open positions, attracting specialized talent by showcasing each department's unique identity.

âš¡ Core Components & Page Breakdown
A. The Header's DepartmentsDropdown.jsx (New or Updated)
Component: src/components/header/DepartmentsDropdown.jsx (or similar)
Description: This component will render the interactive dropdown menu that appears when a user hovers over/clicks the "Departments" link in your main Header.
Content:
A list of clickable links, each representing a department.
Example Links (from mock data):
"Marketing & PR" -> /department/marketing-pr
"Customer Success & Sales" -> /department/customer-success-sales
"IT, Product, Data" -> /department/it-product-data
"Finance & Administration" -> /department/finance-administration
"HR & more" -> /department/hr-more
Styling Notes:
Use Tailwind CSS for dropdown styling (e.g., absolute, bg-white, shadow-lg, rounded-md).
Hover states for individual links.
B. The Dynamic Department Page (src/app/department/[slug]/page.jsx)
Page Path: src/app/department/[slug]/page.jsx
URL Structure Example:
/department/marketing-pr
/department/it-product-data
Description: This is the single page template that will render content for any given department. It will receive the slug (e.g., marketing-pr) from the URL parameters. Based on this slug, it will fetch (from src/data/departments.js for now) the specific content for that department and pass it down as props to the reusable components.
Component Integration: This page will import and use the following reusable components from src/components/department/, passing dynamic props to each:
1. Hero Department Section
Component: src/components/department/HeroDepartmentSection.jsx
Props: departmentData (containing name, intro paragraphs, CTA link, background image path)
Content:
Headline: departmentData.name
Paragraphs: departmentData.introParagraphs
Call-to-Action Button: "See All Openings" (links to /department/[slug]#openings).
Background Visual: departmentData.heroBackground image.
2. Roles & Culture Section
Component: src/components/department/RolesAndCultureSection.jsx
Props: departmentData.rolesAndCulture (an array of role objects, each with title, description, icon)
Content:
Headline: "Our {departmentData.name} Roles & Responsibilities".
Grid of Role Cards: Dynamically map departmentData.rolesAndCulture to display each role.
3. Team Photos Gallery
Component: src/components/department/TeamPhotosGallery.jsx
Props: departmentData.teamGallery (containing main team photo, quotes, office interior photo)
Content:
Headline: "People at DocFinder {departmentData.name}".
Main Team Photo: departmentData.teamGallery.mainPhoto.
Quotes: departmentData.teamGallery.quotes.
Office Interior Photo: departmentData.teamGallery.officePhoto.
4. Testimonial Section
Component: src/components/department/TestimonialSection.jsx
Props: departmentData.testimonial (optional: a specific testimonial for this department, or a generic CTA)
Content:
Headline: "See more job offers in other departments" (A general CTA, or department-specific if testimonial prop has it).
Background: Generic or departmentData.testimonialBackground.
5. Department Positions Section (Job Listings for this Department)
Component: src/components/department/DepartmentPositionsSection.jsx
Props: departmentData.openPositions (an array of job listing objects)
Content:
Headline: "Explore Open {departmentData.name} Positions at DocFinder".
Job Listings: Dynamically map departmentData.openPositions to display available roles.

code
Jsx
// src/app/department/[slug]/page.jsx
import HeroDepartmentSection from '@/components/department/HeroDepartmentSection';
import RolesAndCultureSection from '@/components/department/RolesAndCultureSection';
import TeamPhotosGallery from '@/components/department/TeamPhotosGallery';
import TestimonialSection from '@/components/department/TestimonialSection';
import DepartmentPositionsSection from '@/components/department/DepartmentPositionsSection';
import { getDepartmentBySlug } from '@/data/departments'; // Import your mock data helper

export default function DepartmentPage({ params }) {
  const { slug } = params;
  const departmentData = getDepartmentBySlug(slug);

  if (!departmentData) {
    // Handle case where department is not found (e.g., show 404 or redirect)
    return <div>Department not found.</div>;
  }

  return (
    <main>
      <HeroDepartmentSection departmentData={departmentData} />
      <RolesAndCultureSection departmentData={departmentData} />
      <TeamPhotosGallery departmentData={departmentData} />
      <TestimonialSection departmentData={departmentData} />
      <DepartmentPositionsSection departmentData={departmentData} />
    </main>
  );
}
